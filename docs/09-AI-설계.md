# AI 설계

> 작성일: 2026-07-03
> 관련: [00 §9 계층형 제어](00-개요-및-게임설계.md), [00 §12 전투](00-개요-및-게임설계.md), [05 개발프로세스](05-개발프로세스.md)
> 왜 중요한가: AI는 부가기능이 아니라 **데모를 떠받치는 핵심**이다 — 어트랙트(항상 도는 AI vs AI), 상대역, 빈 슬롯 백필, 그리고 **헤드리스 테스트의 구동체**.

---

## 1. AI = Controller 구현체 (아키텍처 핵심)

AI는 별도 경로가 아니라 **인간 입력·전략 지시와 동일한 `Controller` 인터페이스**를 구현한다([00 §9]). 제어 계층의 어느 층에 꽂히느냐만 다르다:

```
trait Controller {              // 인간/AI/전략이 모두 구현
    fn decide(&mut self, view: &GameView) -> ControlOutput;
}
```
- **AI(전략 층)**: `view`(공·로봇·스코어·시간)를 보고 전략 모드를 고른 뒤, 행동 층(FSM)에 위임.
- **AI(행동/액추에이터 층)**: FSM이 목표를 액추에이터 명령(추력/회전/슛/태클/점프)으로 변환.
- 사람이 슬롯을 잡으면 그 층만 사람 Controller로 교체, 나머지 층은 AI가 유지 → **런타임 전환**이 자연스럽다.

이 통일 덕에: 새 AI 추가 = Controller 구현 추가, **스크립트 봇(고정 입력)** 도 같은 인터페이스라 테스트에 재사용.

## 2. 행동 층 — FSM

```
        ┌────────────── (공 소유/위치·상대 상태로 전이) ──────────────┐
        ▼                                                            │
[Idle] → [ChaseBall] → [DribbleToGoal] → [AlignShot] → [Kick] ───────┤
   │          │              │                                       │
   │          │              └─(상대 근접·공 소유)→ [Evade] ──────────┤
   │          └─(상대가 공 소유)→ [Defend] → [Intercept]/[Tackle] ────┤
   │                                                                 │
   └─(부위 손상·과열, 안전할 때)→ [Rest] ──────────────────────────────┘
```

| 상태 | 하는 일 | 주요 전이 조건 |
|---|---|---|
| Idle | 대기/포지셔닝 | 공이 움직임/경기 시작 |
| ChaseBall | 공으로 이동 | 공 소유 확보 → DribbleToGoal |
| DribbleToGoal | 공 밀며 골대로 | 슛 각/거리 → AlignShot / 상대 근접 → Evade |
| AlignShot | 슛 각 정렬 + 세기 충전 | 정렬 완료 → Kick |
| Kick | 슛 릴리즈 | 후딜 → ChaseBall |
| Defend | 골대·공 사이 차단 위치 | 공 근접 → Intercept/Tackle |
| Intercept/Tackle | 공 가로채기/상대 태클 | 성공/실패 → ChaseBall |
| Evade | 회피 대시로 태클 피함 | 안전 → DribbleToGoal |
| Rest | 정지 회복(스태미나·부위 HP) | 회복/위협 도달 → 복귀 |

> behavior tree는 범위 밖. 위 FSM으로 충분(가볍게 유지).

---

## 3. 전략 층 — 모드 선택

간단한 규칙 기반(휴리스틱). 매 결정 tick(예: 5~10Hz)마다 재평가:
- 공이 **우리 진영** & 상대 소유 → `Defend`
- 공이 **중립/상대 진영** & 우리 소유 → `Attack`(DribbleToGoal)
- 공 미소유 & 가까움 → `Chase`
- **점수 뒤지고 시간 부족** → 공격 가중(aggression↑)
- **부위 손상/과열 심함 & 공에서 멂** → `Rest` 기회 탐색

---

## 4. 전투는 "수단" — 북극성 준수 ([00 §12], [07 ADR-008])

AI도 **축구가 주, 전투는 양념**:
- **태클/넉백은 공 탈취·공간 확보가 목적** — 공에서 먼 상대를 굳이 패지 않음(저가치).
- 상대를 **파손 다운시키면 득점 창** → 그때 공을 골로.
- **도망형 대응**: 손상·과열 시 무리한 교전 대신 회피·정비. 스태미나 관리로 지속 주행.
- 즉 AI 의사결정의 최종 목적함수는 언제나 **"골 기대값"**, 전투는 그 하위 수단.

## 5. 난이도 (설정값)

데이터 주도 파라미터로 조절(하드코딩 X):

| 파라미터 | 의미 |
|---|---|
| `reactionDelay` | 반응 지연(ms) — 클수록 약함 |
| `aimError` | 슛/이동 조준 오차 |
| `decisionHz` | 전략 재평가 빈도 |
| `aggression` | 교전/태클 성향 |
| `staminaCaution` | 과열·손상 회피 성향 |

- 어트랙트는 중간 난이도. 향후 로봇/게임별 프리셋 가능.

---

## 6. 결정성 & 테스트 (핵심)

- **결정적**: AI 의사결정에 **시드 없는 난수 금지**. 필요 변화는 시드 고정. → 리플레이·회귀 테스트 보존([07 ADR-007]).
- **스크립트 봇**: 고정 입력 시퀀스를 내는 `Controller` 구현 → 게임 로직을 **결정적으로 유닛/통합 테스트**([05](05-개발프로세스.md)).
- **헤드리스 AI 대전**: 렌더 없이 서버 sim만으로 AI vs AI를 돌려 **성능 벤치·안정성·밸런싱** 확인.
- FSM 전이·전략 선택은 **순수 함수 → TDD 최적**.
