# 개발 프로세스 (TDD + 칸반)

> 작성일: 2026-07-03
> 관련: [00-개요-및-게임설계](00-개요-및-게임설계.md), [03-개발환경](03-개발환경.md)
> 방식: **TDD(테스트 주도 개발)** + **칸반 보드([../KANBAN.md](../KANBAN.md))** 순서 진행. 최종적으로 전체 유닛 테스트 스위트 확보.

---

## 1. 원칙

- **테스트 먼저**: 각 태스크는 실패하는 테스트부터 작성(Red) → 최소 구현(Green) → 리팩터(Refactor).
- **결정적 코어**: 게임 로직을 I/O(소켓·렌더)에서 분리. "입력 → tick → 결정적 상태"가 되도록 난수/시간 의존 최소화(시드·고정 timestep). → 소켓 없이 경기 전체를 테스트 가능.
- **작은 카드**: 칸반 카드 하나 = TDD 사이클 하나 정도의 작은 단위.
- **칸반 순서 준수**: Backlog에 미리 등록된 카드를 위에서부터 순서대로 진행. WIP는 1(한 번에 한 카드)을 기본.

---

## 2. TDD가 잘 맞는 영역 / 아닌 영역

| 영역 | TDD | 테스트 방법 |
|---|---|---|
| 게임 규칙(스코어·3분 종료·승/무/패) | ✅ | 순수 로직 유닛 |
| 제어 FSM(공추적→정렬→슛, 수비) | ✅ | 상태·전이 유닛 |
| 스태미나/오버히트/디버프/쿨다운 | ✅ | 입력→상태 유닛 |
| 랭킹 승률 집계 | ✅ | 순수 함수 유닛 |
| 프로토콜 (de)serialize | ✅ | serde 라운드트립 |
| 재연결/슬롯 인계 상태머신 | ✅ | 로직만(소켓 없이) |
| 물리(rapier) 결과 | ⚠️ 부분 | **불변식**만: 공 경기장 이탈 없음, 골 라인 통과 시 감지, 충돌 후 속도 방향 등. 정확한 float 값은 테스트 안 함 |
| 렌더/이펙트/보간 시각결과 | ❌ | 수동/시각 확인. 보간 수학·데이터만 유닛 |
| 조작감·밸런싱 튜닝 | ❌ | 플레이하며 조정 |

## 3. 테스트 도구 & 종류

### 서버 (Rust) — `cargo test`
- **유닛**: 모듈 내 `#[cfg(test)]` — FSM, 스태미나, 디버프, 랭킹, 메시지 직렬화.
- **통합**: `server/tests/` — 경기 루프를 입력 메시지 시퀀스로 구동해 스코어/상태 검증.
- **물리 불변식**: rapier 월드를 N tick 돌린 뒤 경계·골 감지 등 성질 단언.

### 클라 (TS) — `vitest`
- 메시지 파싱/타입, 보간 수학, 클라 상태머신(ATTRACT/SELECT/PLAYING/RESULT) 전이.
- canvas 렌더 자체는 유닛 대상 아님(로직·데이터만).

### (선택) CI
- 매 커밋/PR에서 `cargo test` + `vitest` 실행. GitHub Actions 등.

## 4. TDD 사이클 (카드 단위)

```
Backlog에서 카드 하나 → In Progress
  1. Red     : 원하는 동작을 실패 테스트로 명시
  2. Green   : 통과할 최소 코드
  3. Refactor: 초록 유지하며 정리
  → 테스트 통과 확인 → 커밋 → 카드 Done
```

- 커밋 메시지에 카드 참조(예: `[KB-12] 스태미나 소모/회복`).
- "Done"의 정의(Definition of Done): 관련 테스트 통과 + 커밋 완료 + (해당 시) 문서 갱신.

## 5. 칸반 운영 ([../KANBAN.md](../KANBAN.md))

- 컬럼: **Backlog → Todo → In Progress → Done**.
- 카드 형식:
  ```
  - [ ] KB-NN 제목 — 한 줄 설명 (테스트: 무엇을 검증) [의존: KB-xx]
  ```
- Backlog는 구현 계획(writing-plans 산출)에서 **의존성 순서**로 채운다.
- 진행: 맨 위 카드부터 Todo → In Progress(WIP 1) → Done.
- 대략적 진행 순서(에픽): 프로젝트 스캐폴딩 → 메시지 스키마 → 결정적 sim 코어 → 게임 규칙 → 제어/FSM → 액션/디버프 → 전투/데미지 → 네트워크 세션/재연결 → 클라 렌더/보간 → 아이템 → 랭킹 → 무선 시뮬레이터 → 폴리싱.

## 6. 테스트 아키텍처

| 스위트 | 대상 | 방법 |
|---|---|---|
| **유닛(순수 로직)** | 게임 규칙·FSM·전략·스태미나·디버프·전투 데미지/효과 선택·랭킹·(de)serialize | 순수 함수 in/out. 결정적 |
| **물리 불변식** | rapier 월드 | N tick 구동 후 **성질** 단언: 공 경기장 이탈 없음, 골 라인 통과 시 감지, 충돌 후 속도 방향, 파손 다운→리페어 복귀. **정확한 float 값은 단언 안 함** |
| **통합(헤드리스 sim)** | 경기 루프 전체 | **스크립트 Controller**(고정 입력)로 경기를 소켓 없이 구동 → 스코어·상태·전투 결과 검증 ([09 §6](09-AI-설계.md)) |
| **골든 리플레이(결정성 회귀)** | 결정성 | 기록된 입력·시드로 재생 → **스냅샷 해시 비교**. 결과가 바뀌면 회귀 감지 (same-build, [07 ADR-007](07-결정기록-ADR.md)) |
| **네트워크** | 프로토콜/세션 | **목(mock) 전송**으로 메시지 왕복·재연결/슬롯 인계·미지 `t` 무시 검증 |
| **클라(vitest)** | 메시지 파싱·보간 수학·상태머신 전이 | canvas 렌더는 제외(로직만) |

- **결정적 코어 + I/O 분리**가 위 대부분을 가능케 함(소켓·렌더 없이 테스트).
- CI: 매 커밋 `cargo test` + `vitest` (+ 골든 리플레이).

## 7. 완료 정의 (DoD)

**카드 단위 DoD**: 관련 테스트 통과 + 커밋 + (해당 시) 문서 갱신.

**v1(코어) 완료 정의** — 아래를 모두 만족하면 v1 "done":
- [ ] 어트랙트에서 AI vs AI 경기가 끊김 없이 라이브로 돎
- [ ] 브라우저 2탭(또는 폰)으로 각 슬롯 참가 → 1:1 실시간 경기 성립
- [ ] 직접/전략/AI **런타임 전환** 동작
- [ ] 3분 경기 → 골로 승/무/패 판정, 로봇 승률 랭킹 갱신
- [ ] 전투: 상호 데미지·부위 HP·파손 다운→리페어·넉백/스턴, 공 탈취
- [ ] 파츠 로드아웃으로 스탯 달라짐(커스터마이즈)
- [ ] 끊김 시 재연결/슬롯 AI 인계
- [ ] NET SIM으로 지연·드랍 걸어도 보간으로 부드러움
- [ ] 골든 리플레이 회귀 통과 + 전체 유닛 스위트 그린
- [ ] README + 실행 GIF (30초 안에 이해)
