## 1. 상태 머신 (전체 흐름)

클라이언트는 SPA 하나로, 서버가 보내는 경기 상태(`phase`)에 따라 아래 화면을 전환한다.

```
   ┌──────────────────────────────────────────────┐
   ▼                                               │
[ATTRACT]  AI vs AI 라이브 데모 (중앙 "DEMO" 표시)     │
   │  아무 키 / 슬롯 클릭                            │
   ▼                                               │
[ROBOT SELECT]  로봇 선택 (8칸, 2 오픈 / 6 잠금)      │
   │ Enter=선택·참가          │ Esc=취소 ────────────┘
   ▼                          ▲ (되돌아감)
[PLAYING]  3분 경기 (직접/전략/AI 제어, 런타임 전환)
   │ 경기 종료
   ▼
[RESULT]  승 / 무 / 패 + 보상 → 로봇별 승률 랭킹 갱신
   │ 재대결 → PLAYING     │ 로비/타임아웃 → ATTRACT
   └──────────────────────┘
```

**어트랙트 모델의 핵심**: 서버에는 1:1 경기가 **항상 진행 중**이다. 두 슬롯(BLUE/RED)은 기본 AI가 채우며, 사람은 빈자리에 들어가는 게 아니라 **AI가 몰던 로봇을 이어받는다**. 나가면 AI가 다시 인계 → 자연스럽게 DEMO로 복귀. 방/코드/카운트다운 같은 로비 군더더기가 없다.

## 2. 화면별 목업

### 2.1 ATTRACT (대기 = 라이브 AI 데모)

```
┌───────────────────────────────────────────┐
│  BLUE 🤖 0:0 🤖 RED  ⏱ --:--        ≡ 메뉴  │
│   메뉴: 커스터마이즈 · 전적·보상 · 샵🔒 · 메일🔒 │
│                                             │
│      □ ───→ ○공        ■        ← 라이브 경기 │
│              ★ DEMO ★         ← 중앙 오버레이 │
│   [ BLUE로 참가 ]      [ RED로 참가 ]        │
│              ● 서버 연결됨                    │
└───────────────────────────────────────────┘
```
- 중앙 "DEMO" 오버레이는 **아무도 제어 안 할 때만** 표시. 슬롯을 잡으면 사라짐.
- 접속 즉시 상태 수신(관전) 시작 → "이 시스템이 지금 실제로 돌고 있다"를 증명.

#### 메뉴 구성 (정보 구조 · IA)

메뉴 자리는 **지금 확정**하되, 미구현 항목은 **"준비 중" 비활성(🔒)** 으로 표기(기존 로봇/파츠 잠금 패턴과 동일).

| 메뉴 | 상태 | 대응 페이즈 |
|---|---|---|
| 로봇 선택·참가 | ✅ | v1 |
| 커스터마이즈 — **장착(파츠)** | ✅ | v1 (스탯·외형) / 소유·저장은 Phase 2 |
| 커스터마이즈 — **꾸미기(코스메틱)** 🔒 | 준비 중(초기 죽은 버튼) | v1 후반 (스탯0 파츠) |
| 전적 · 보상(랭킹) | ✅ | v1 (인메모리) |
| **샵** 🔒 | 준비 중(죽은 버튼) | **Phase 3** (인게임 재화 구매) |
| **메일** 🔒 | 준비 중(죽은 버튼) | **Phase 4** (보상·알림 수령) |

- 장착(파츠)·꾸미기(코스메틱) 메뉴는 커스터마이즈 화면([§2.2](#22-robot-select-캐릭터-셀렉트))의 두 탭으로 진입. 둘 다 경기 참가 전에 접근.
- IA를 지금 고정 → 나중에 항목이 채워져도 레이아웃 재작업 없음.
- **UI 방침(채택)**: 미구현 항목은 **"준비 중" 죽은 버튼**으로 노출(로드맵 신호). 명확히 "준비 중" 표기라 미완성으로 보이지 않게. 페이즈 정의는 [00 §14 확장 로드맵](00-개요-및-게임설계.md)·[07 ADR-005/006](07-결정기록-ADR.md).

### 2.2 ROBOT SELECT (캐릭터 셀렉트)

```
┌────────────────────────────────────────────┐
│               ROBOT SELECT                    │
│  ┌──────────────────────────────────────┐   │
│  │   [ 선택 로봇이 실제로 움직이는 프리뷰 ] │   │ ← 위: 움직임 애니메이션
│  └──────────────────────────────────────┘   │
│  ◀ [🤖1][🤖2][🔒][🔒][🔒][🔒][🔒][🔒] ▶     │ ← 8칸(2 오픈 / 6 잠금)
│  ┌──────────────────────────────────────┐   │
│  │ 🤖 STRIKER-01                         │   │ ← 아래: 이미지 + 설명
│  │ 가속 빠름 · 몸싸움 약함 · 슛 강함        │   │
│  │ 승률 62%  (12승 5무 8패)              │   │
│  └──────────────────────────────────────┘   │
│    ← → 이동    Enter 선택·참가    Esc 취소     │
└────────────────────────────────────────────┘
```
- 위=선택 로봇의 실제 움직임 프리뷰, 아래=이미지+설명(스탯 성향)+승률.
- 잠금 칸(🔒)="Coming soon" 표시만. 로봇 추가 시 자동 노출.
- 로스터는 **승률 높은 순** 정렬 표시.
- 프리셋은 **파츠 조립체**([00 §6](00-개요-및-게임설계.md)). `[커스터마이즈]` 진입 시 슬롯(머리/목/몸통/앞다리2/뒷다리2/꼬리)별로 해금 파츠를 교체 → 총 스탯·외형 변경.

```
┌ CUSTOMIZE ─────────────────────────────────┐
│  [ 장착(파츠) ] [ 꾸미기(코스메틱) 🔒 ]        │ ← 탭
│  [ 조립된 로봇 프리뷰 + 실시간 스탯 막대 ]     │
│  머리   [ 표준 ▼ ]   목    [ 표준 ▼ ]        │
│  몸통   [ 경량 ▼ ]   앞다리 [ 파워 ▼ ]        │
│  뒷다리 [ 스피드 ▼]  꼬리   [ 밸런스 ▼]        │
│    (잠금 파츠는 🔒 비활성)                     │
│  [ 저장·선택 ]        [ 뒤로 ]                 │
└─────────────────────────────────────────────┘
```
- 파츠 교체 시 스탯 막대가 즉시 갱신 → 조합 결과를 눈으로 확인.
- **결제/가챠/영속 저장 없음**: 로드아웃은 세션/해금 기반. 근거 [06 §3](06-포트폴리오-전략.md).

### 2.3 PLAYING (경기 중 — 핵심 화면)

```
┌───────────────────────────────────────────────┐
│ [YOU: BLUE]    2 : 1    [RED]        ⏱ 01:23    │ ← HUD: 팀/스코어/타이머
│ ● connected   tick 30/s   ping 12ms            │ ← 링크 품질계기판
├───────────────────────────────────────────────┤
│   ▐                                       ▌     │
│ 골 ▐        □(나)      ○공        ■(상대)  ▌ 골  │ ← 탑다운 2D canvas
│   ▐               ◇아이템                  ▌     │   (서버 상태 렌더)
├───────────────────────────────────────────────┤
│ 제어 모드:  [ 직접 ] [ 전략 ] [ AI ]            │ ← 모드 스위처(런타임 전환)
│ ┌ (선택 모드에 따라 패널 내용이 바뀜) ─────────┐ │
│ │ 직접 : ↑이동 ←→회전 Space슛 Shift달리기      │ │
│ │        X태클 Z점프 S스톱 R쉬기               │ │
│ │ 전략 : [공격][수비][공추적][대기] / 필드클릭  │ │
│ │ AI   : 🤖 자동 플레이 중…                    │ │
│ └───────────────────────────────────────────┘ │
│ 스태미나 [████████░░]   슛세기 [▓▓▓░░]          │ ← 자원 게이지
└───────────────────────────────────────────────┘
```
- 로봇 위에 상태 표시: 스턴(별), 오버히트(글로우), 쿨다운(작은 게이지).
- 모드 스위처로 자기 로봇을 `직접↔전략↔AI` 런타임 전환.

### 2.4 RESULT (경기 후)

```
┌────────────────────────────┐
│         경기 종료           │
│       BLUE  2 : 1  RED      │
│        🏆 BLUE 승리          │
│        보상  +50 코인        │
│   [ 재대결 ]   [ 로비로 ]    │
└────────────────────────────┘
```
- 무승부 시 "무승부" 표시. 결과가 해당 로봇의 승률 랭킹에 반영.

## 3. 플레이 방식 & 조작 요소 (How to Play)

### 3.0 한눈에 보는 플레이 흐름
1. 접속하면 **ATTRACT**에서 AI끼리의 라이브 경기를 관전한다.
2. **[BLUE로 참가]/[RED로 참가]** 또는 아무 키 → **로봇 선택**.
3. 로봇을 골라 참가하면 그 팀 로봇을 이어받아 **경기(3분)** 시작.
4. **골대에 공을 더 많이 넣으면 승리.** 종료 시 승/무/패.
5. 경기 중 **제어 모드(직접/전략/AI)를 자유롭게 전환**하며 플레이.
6. 종료 → 결과·보상 → 재대결 또는 ATTRACT 복귀. 결과는 로봇 승률 랭킹에 반영.

> 목표: 3분 안에 상대보다 많은 골. 공은 물리로 움직이며, 로봇으로 밀고/차고/태클해 다룬다.

### 3.1 화면별 버튼

**ATTRACT**
| 버튼 | 사용법 | 결과 |
|---|---|---|
| [BLUE로 참가] / [RED로 참가] | 클릭(또는 아무 키) | 로봇 선택 화면으로 |

**ROBOT SELECT**
| 요소 | 사용법 | 결과 |
|---|---|---|
| ◀ ▶ (또는 ← →) | 클릭/키 | 선택 로봇 좌우 이동 |
| 로봇 칸 8개 | 클릭 | 로봇 선택(잠금 칸은 비활성) |
| [선택·참가] (Enter) | 클릭/Enter | 그 로봇으로 참가 |
| [취소] (Esc) | 클릭/Esc | ATTRACT 복귀 |

**PLAYING (항상 표시)**
| 버튼 | 사용법 | 결과 |
|---|---|---|
| [직접] [전략] [AI] 모드 스위처 | 클릭 | 내 로봇 제어 방식 런타임 전환 |
| [나가기] | 클릭 | 슬롯 반납 → AI 인계, ATTRACT 복귀 |

**RESULT**
| 버튼 | 사용법 | 결과 |
|---|---|---|
| [재대결] | 클릭 | 같은 로봇으로 다시 |
| [로비로] | 클릭 | ATTRACT 복귀 |

### 3.2 경기 중 모드별 조작

**직접 모드 (데스크톱 = 키보드)**
| 키 | 액션 |
|---|---|
| ↑ / ↓ | 전진 / 후진 |
| ← / → | 좌 / 우 회전 |
| Shift (홀드) | 달리기 |
| S | 스톱(제동) |
| R (홀드) | 쉬기 |
| Space (홀드→릴리즈) | 슛(누른 만큼 세기 충전) |
| X | 태클 |
| Z | 점프(회피 대시) |

**전략 모드 (마우스 / 터치)**
| 요소 | 사용법 | 결과 |
|---|---|---|
| 필드 클릭/탭 | 지점 지정 | 그 지점으로 이동 지시 |
| [공격] [수비] [공추적] [대기] | 클릭/탭 | 상위 전략 지시(행동층 FSM이 수행) |

**AI 모드**: 조작 없음. "🤖 자동 플레이 중" 표시만.

### 3.3 입력 수단 매트릭스
| 기기 | 메뉴 조작 | 직접 제어 | 전략 제어 |
|---|---|---|---|
| 데스크톱 | 마우스/키 | 키보드 ✅ | 마우스 + 버튼 ✅ |
| 폰(확장) | 터치 | ❌(키보드 없음) | 터치 탭 + 버튼 ✅ |

- 게이지(스태미나·슛세기)와 상태 아이콘(스턴/오버히트/쿨다운)은 **버튼이 아니라 정보 표시**.
- NET SIM 패널(지연/지터/드랍)은 플레이어용이 아니라 **개발/시연용 오버레이**([02](02-네트워크-프로토콜.md) §7).

> 키 배치는 잠정. 구현 중 조작감에 따라 조정.

## 4. 제어 모드 전환 UX

- 하단 **모드 스위처**(직접 / 전략 / AI) 토글로 경기 중 즉시 전환.
- 전환 시 서버로 `ctrlmode` 메시지 전송([02](02-네트워크-프로토콜.md) 참조), 서버가 해당 층부터 사람/AI 제어를 스위칭.
- 시연 포인트: 같은 로봇을 세 방식으로 몰아 보이기.

---

## 5. 랭킹 표시

- **로봇 캐릭터별 승률**을 높은 순으로 표시(로봇 선택 화면 카드 + 별도 랭킹 목록).
- 인메모리 집계, 서버 재시작 시 리셋([00](00-개요-및-게임설계.md) §5).

---

## 6. 모바일 / 터치 UX (확장)

- 웹 클라이언트라 **폰 브라우저로 접속 가능**.
- 폰엔 키보드가 없으므로:
  - **관전(DEMO 보기)**: 그대로 지원.
  - **플레이**: **전략 모드(터치)** — 필드 탭=이동 지시, 버튼=공격/수비/슛.
- **크로스 디바이스 시연**: 데스크톱(키보드=직접) vs 폰(터치=전략) 실시간 대결 → "무선 링크 + 다양한 제어" 어필.
- 요구사항: **반응형 레이아웃** + 터치용 전략 조작 UI.
- 우선순위: **데스크톱 우선**, 폰/터치는 이후 확장(웹이라 나중 추가 무리 없음).

## 7. 이펙트 / 애니메이션 (다듬기 영역)

- 로봇 = 사각형 프리미티브(몸통+다리+목+머리). 포인트는 **이동/방향전환 애니메이션**.
- 슛/태클/점프/스턴/오버히트/아이템 획득에 시각 이펙트.
- 이 영역은 구현하며 반복 조정(핵심 로직과 분리된 폴리싱 작업).
