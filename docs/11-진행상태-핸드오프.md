## 1. 이게 뭔가 (한 줄 논지)

**1v1(팀당 2대) 로봇 축구 실시간 웹 데모.** 어필 포인트는 게임성이 아니라
**실시간 통신 + 계층형 로봇 제어(사람/AI) + 서버 권위 물리 + 결정적 코어**.
로봇 축구는 그것을 "가볍게" 담는 그릇. 포트폴리오 조각.

- **서버 권위**: 물리·판정·전투·스코어 전부 서버(Rust). 클라는 렌더+입력만.
- **결정적 코어**: 순수 `step`/`tick`(I/O·RNG·HashMap순회 배제) + 고정 timestep + 골든 리플레이.
- **계층형 제어**: `Controller` 트레잇으로 사람/AI를 런타임 스왑(같은 슬롯을 사람↔AI로 교체).

## 2. 현재 상태 (플레이 가능한 슬라이스)

**팀당 2대(공격형 striker + 방어형 guard), 총 4대**가 경기하는 완성된 코어 루프:
이동·스프린트·**차기(슛)**·전투(HP/스턴/넉백/파손다운)·골/스코어·**협동 AI(공격/수비 역할분담)**·
사람 참가(팀 striker 조종)·넷코드(보간/링크심/RTT)·다듬어진 콘솔 UI.

- 서버 테스트 **110개 통과**, debug+release **경고 0**. 클라 `npm run build` clean.
- 미완성은 "기능"보다 **포장/흐름**(아래 §7): 게임 흐름 화면·랭킹·README/GIF 등.

## 3. 기술 스택 · 실행

| | |
|---|---|
| 서버 | Rust — rapier2d 0.26(물리, enhanced-determinism), tokio, axum(WS), serde. `0.0.0.0:8090` 바인드 |
| 클라 | Vite + vanilla-TS, Canvas 2D, WebSocket(`ws://127.0.0.1:8090/ws`) |
| 통신 | WS + JSON. 서버 60Hz 물리 / 30Hz 스냅샷 브로드캐스트. 클라 보간(예측 없음) |

```bash
# 서버 (server/)
cargo run            # 8090 바인드. ★서버 코드 변경 시 반드시 재시작
cargo test           # 110 통과
# 클라 (client/)
npm install && npm run dev   # 렌더/UI는 핫리로드(재시작 불필요)
```
- Windows에서 서버 실행 중이면 `target/debug`가 잠기므로, 별도 빌드는
  `CARGO_TARGET_DIR=…/target-verify`로 격리(그 폴더는 gitignore됨).

## 4. 아키텍처 맵

### 서버 `server/src/`
- **world.rs** — 타입/상수. `MATCH_KICKOFF[4]`(4대 로스터 배치), `RobotState`(id=Team, pos/rot/vel/parts/down/st/stamina/robot), `GameState`(robots/ball/score/time/**ctrl[]**), `ControlOutput`(thrust/turn/run/kick), 상수(FIELD_W=12, FIELD_H=8, GOAL_W=2.4, DT=1/60).
- **physics.rs** (최대) — `PhysicsWorld`: rapier 월드(벽/골펜스/코너챔퍼/공/로봇), `teams[]`·`kickoff_layout`, `new_match()`(4대·실행용)·`new_kickoff_with([;2])`(2대·테스트용, `#[cfg(test)]`), 공용 `build()`. `apply_controls`(이동·스태미나·maxSpeed클램프)·`apply_kicks`(상승엣지 슛)·`clamp_ball_speed`·`step`·`check_goal`(+이탈 안전망)·`snapshot`.
- **control.rs** — `Controller` 트레잇. `ChaseBallAi`(공격: 추적+슛+스턱탈출), `DefenderAi`(수비: 골-공 선상 지킴+도착 시 공조준 대기+클리어). 공용 함수(enemy/own_goal_x, wants_kick, escape).
- **combat.rs** — 순수 데미지/효과(넉백/스턴/데미지)·부위HP·파손다운.
- **parts.rs** — 파츠/스탯 카탈로그(`StatSet`), 프리셋 striker/guard, `aggregate()`.
- **stamina.rs** — 순수 스태미나(drain/regen/ratio).
- **session.rs** — 업링크 파싱(join/input/leave). **net.rs** — WS 핸들러 + 연결별 **링크심(지연/지터/드랍)**·ping→pong·netsim 파싱. **main.rs** — sim 태스크(new_match+watch), `SlotControllers`(4슬롯: 0/2=공격 AI, 1/3=수비 AI; 팀 join→그 팀 striker를 사람; AFK 30초 해제), ctrl 방출.
- **loop_runner.rs**(tick), **accumulator.rs**(고정스텝), **replay.rs**(골든 리플레이).

### 클라 `client/src/`
- **net.ts** — WS 연결/송신, 스냅샷 버퍼, ping/RTT, netsim, mySlot 추적. **interp.ts** — 보간(인덱스 매칭·최단호). **render.ts** — Neon Telemetry Arena 캔버스(팔각 경계·팀 글로우 6족 로봇·발자국 링·공 글로우·HP/스태미나 캡슐). **hud.ts** — 스코어/시계/참가 배지(사람/AI/YOU). **devpanel.ts** — netsim 슬라이더·보간 토글·RTT. **input.ts** — 키보드·참가 토글. **index.html/style.css** — 미드나잇 프리시전 콘솔 레이아웃.

## 5. 완료 기능 (커밋 그룹)

- **Plan 1~4a**(KB-01~42): 결정적 뼈대 → rapier 물리 → 파츠/스탯 → 전투/데미지 → 효과 → 사람 조작 최소 슬라이스.

- **Plan 7a 넷코드**(KB-50): 보간 + 서버 링크심 + ping/RTT.
- **UI 디자인**(KB-51): 미드나잇 콘솔 + Neon Telemetry Arena(ADR-014, 독립 리뷰어 2패널로 결정).
- **슬롯 UX**(KB-55): 참가 토글 + AFK 자동해제 + 상태 배지.
- **팀 로스터/협동**(KB-57): 팀당 2대(공격+방어) + 협동 AI(역할분담) + 친선 데미지 금지(ADR-015).
- **버그픽스**: KB-58 공 필드 이탈(강킥 터널링 → kick_power 재조정·속도상한·CCD·이탈 안전망), KB-59 방어형 제자리 스핀(도착 시 공조준 대기+회전 데드존).
- **수비 AI 방어범위 확대+조기인지**(KB-60): `guard_target`에 공 속도 예측(`LOOKAHEAD`=0.35s)으로 조기 인지 + 예측 위치 진영에 따라 가드 거리 2단화(`GUARD_HOME`=2.5 상대 진영 / `GUARD_ENGAGE`=4.0 자기 진영, 기존 `DEFENDER_GUARD_DIST`를 `GUARD_HOME`으로 개명).
- **자살골 조사+수정**(KB-61~63): (61) `guard_target`의 LOOKAHEAD 외삽이 공 속도의 접선 성분이 클 때 목표를 공의 실제 위치보다 골에서 더 멀게 만드는 오버슈트 수정. (62) 자기 골 1.5m 이내 접근 시 추력 감속(`own_goal_caution_scale`). (63) 물리 레이어에 `last_ball_toucher`(rapier 충돌 이벤트 기반)를 추적해 그 로봇의 자기 골 쪽 속도 성분을 제거 — 최초엔 "로봇-공 현재 근접"으로 게이팅했으나 실측 재현(디플렉션은 로봇과 공이 떨어진 뒤에도 여세로 골까지 굴러가는 경우가 대부분)에서 효과 없음을 확인, "마지막 접촉자" 기준으로 재설계해 해결. 30분 헤드리스 실측(정확한 충돌 이벤트 기반 지표): 자책골 비율 58%→19%.

## 9. 개발 방식 메모
TDD(테스트 먼저) + KANBAN 순서. 실행은 오케스트레이터(상황관리)가 저비용 모델 에이전트에
분담(신규/수정 → 검증 → 최종정리)하는 방식으로 진행함. 버그는 **헤드리스 재현 테스트**로
데이터부터 확인(예: KB-58은 4대 AI 30초 구동 테스트로 공 이탈을 재현·수정). 상세 프로세스는
[05-개발프로세스.md](05-개발프로세스.md)·[10-점검-프롬프트.md](10-점검-프롬프트.md).
